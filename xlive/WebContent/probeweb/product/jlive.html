開發雲端應用系統的全功能平台

  • 從系統設計、流程、除錯到報表作業、系統導入，jLIVE Builder™ 完整的解決方案，讓開發者能隨時為因應多變的商務需求，快速地開發企業應用程式。
  • 多層式架構 (N-tiers architecture)：客戶端層 (client tier) 、中間層 (middle tier) 及資料庫伺服器 (database server tier) 即為最基本的三層式架構。jLIVE™ middle ware 是純 Java™ 的 servlet 程式，不但適用於各種作業平台，亦可安裝於各家知名的應用程式伺服器 (application server) 。

多平台、多資料庫 一次搞定

  • 跨作業平台：網際網路的應用程式，通常是由各種作業平台 (OS) 的瀏覽器所執行，所以企業開發應用程式的首要條件，必然是要能使應用程式在各種不同的作業平台上執行。 jLIVE Builder™ 所使用的 Java™ 具備跨平台的特性，中間層 (middle tier) 亦由 Java™ Servlet 所設計成，所以在各種作業平台 (MS Windows 、Solaris 、Red hat 、MacOs 、Unix ‧‧‧) ，皆能運作。
  • 異質型資料庫系統同時存取 (Heterogeneous database manipulate simultaneously)：網際網路的企業應用整合 (EAI) ，應用程式所面對的可能是企業體中舊有或新增的各種不同資料庫系統，應用程式不只是要能存取各種不同的資料庫，亦要能同時處理各種不同資料庫的資料處理。因此應用程式中間層 (middle tier) 在服務客戶端 (client) 時必須能夠跨越不同的資料庫系統存取、整合資料。 jLIVE Builder™ 的 middle ware 正是支援同時存取多個不同資料庫系統的中間層 (middle tier) 。

內建全功能報表系統 不需另外購置

  • 報表作業是企業應用程式不可或缺的系統服務，報表系統使用上須滿足以下的最低需求：分頁的機制、圖檔及統計圖表列印、套表列印、即時的統計分析列印
  • 報表系統上更必須符合，客戶端 (client) 負責資料運算 (data aggregate) 、統計圖表製作 (statistic chart plot) 、 列印格式配置 (format layout) ，伺服器端 (server) 只按照客戶端 (client) 目前列印需求，提供查詢結果集 (query result set) 的資料，如此的分散運算各司其職，網際網路報表作業才能順利進行。

多國語言自動切換

  • jLIVE Builder™ 不但開發平台為多國語言版，設計師運用 jLIVE Builder™ 亦能輕易地開發多國語言網際網路應用程式。多國語言網際網路應用程式，能根據執行期客戶端 (client) 作業系統語系 (OS locale) 自動轉換對應語系。設計師可以設計一應用程式放諸四海皆準。
  • 異地應用伺服器同時存取：跨國性與多分公司的企業，資料庫系統通常是獨立存在於各據點，網際網路應用程式不但要能處理單一據點資料處理，還要能適時地同時存取各據點的資料訊息。 jLIVE Builder™ 允許在開發應用程式時，同時連接異地的應用伺服器，並存取異地應用伺服器所連結的資料庫系統，跨國性與多分公司的企業，隨時可透過應用程式，統計、分析、處理各據點資料。

不必學習JAVA語言. 快速上手開發

  • 快速的視覺化開發環境：程式設計師可以在完全視覺化的設計環境中，隨時設計，馬上執行，不但使程式的流程更正確，設計的品質亦相對提高。系統以高效率的 Java™ Swing components 元件連結資料，元件內部以 Java™ MVC (Model, View, Controller) 技術運作資料，jLIVE Builder™ 讓每一個程式設計師所設計出的程式，皆是 Java™ 最頂級的技術實現。
  • 一個應用系統多種執行方式：由瀏覽器啟動執行(browser enabled HTML)及以 Java web start™ 啟動執行 (web start JNLP) 的程式，使用者 (user) 可以在任何的作業系統，以瀏覽器或 Java web start™ 透過網際網路或內部網路執行。程式開發者只要在伺服器 (web server) 上維護最新版的應用程式即可，由於 Java™ class Loader 會動態的只下載第一次參照或有更新的 class ，因此不但使用者端能呈現最佳的效率，開發者對程式的維護也最為容易。
  • 物件導向和事件驅動的設計理念：jLIVE Builder™ 的開發環境，是以物件導向 (OOP) 為應用系統開發的核心架構。程式設計師以物件 (object) 架構應用系統，在每個物件上，處理由物件所驅動的事件程序 (event) ，事件程序即是所謂的使用者邏輯 (user logic) 和商業資料邏輯 (business logic) 。設計一個流暢的使用者邏輯，規劃正確且嚴謹的商業資料邏輯，就能開發出擁有親切使用者介面及成功的企業應用系統。在 jLIVE Builder™ 的開發環境中，從資料、元件、視窗畫面、報表以至於事件程序，都成功的整合為視覺化物件，開發者只要以物件導向和事件驅動的設計理念，就可以快速 的設計出應用系統，維護及更新程式也更為容易。

分散式運算 運作效能高

  • 即時、互動的資料庫操作：所謂的分散式運算，即是系統將使用者的操作需求，分別由客戶端 (client tier) 、中間層 (middle tier) 及資料庫伺服器端 (database server tier) 按最有效率的方式分配擔任運算角色。例如：使用者輸入資料的檢核，只需由客戶端 (client) 運算與回應，不需伺服器 (web) 的參與。複雜的報表作業，則由客戶端 (client) 發出列印資料的請求，中間層將請求的資料庫 SQL 命令，轉請資料庫伺服器服務，並傳回結果的資料集 (result set) 。中間層則按客戶端目前印表所需的資料陸續傳至客戶端，至於報表的資料擺設 (layout) 、格式 (format) 、彙整 (aggregate) 或統計圖表，則由客戶端即時運作。此一分散式的運算，不但使用者可以得到最快速的回應，更可以提高伺服器端的服務效率。這也是 jLIVE Builder™ 所開發的應用程式系統，在執行的效率上，快速無比的原因之一。
