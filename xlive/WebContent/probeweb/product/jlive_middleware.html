jLIVE Builder™  所開發的網際網路應用程式是以三層式的架構執行程式，使用者端 (client) 不允許也不可以直接存取後端資料庫系統 (database server) 和檔案系統資源 (file system resource)，所有的存取必須透過中間層 (middle tier) 的應用伺服器 (application server) 取得服務。jLIVE Builder™ 的中間層 (middle tier) 是一個標準的 Java Sevlet™ 模組，可以掛載於任何符合 Java™ J2EE 規格的應用伺服器 (application server) ，例如：IBM Web sphere™、 Web logic ™、 Fujitsu Interstage™、 Tomcat™ ‧‧‧。與應用伺服器 (application server) 協同合作處理網際網路的 Web 伺服器服務 (Web service)。

jLIVE Builder™ 的中間層 (middle tier) 是由 pure Java™ 所撰寫的 Servlet 模組 -- jslive.jar (以下簡稱 jLIVE ™ middle ware)，以 JAR 檔的型式掛載於應用伺服器，在應用伺服器啟動時由應用伺服器啟動服務，並納入應用伺服器的 Servlet 管理，與一般應用伺服器上的 Servlet 或 JSP 並沒有分別，只是 jLIVE™ middle ware 專屬提供 JLIVE 應用程式使用者端的服務請求 (service request)。

 jLIVE™ middle ware 除了提供使用者端的服務請求外，亦提供 jLIVE™ middle ware 的管理介面，透過管理介面，應用伺服器管理員可以掌控應用程式執行的狀態，即時監控使用者連結及取用資訊的情況。
