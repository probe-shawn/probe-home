分店資訊 (store) 的表格架構

back

企業代碼 (cid)

公司加入 yup 時，必須具備一個 Facebook 粉絲頁帳號，其 Facebook id 即是企業代碼，不可自行更改。

分店代碼 (sid)

分店加入 yup 時，必須具備一個 Facebook 粉絲頁帳號，其 Facebook id 即是分店代碼，不可自行更改。

店舖名稱 (store_name)

必要欄位，不得空白。

次名或簡稱 (sub_name)

非必要欄位。

店舖簡述 (store_desc)

店舖歷史、信念、專長、沿革、特色等等資訊，此欄位內容會顯示在分店首頁。

店長fid (fid)

店長的 Facebook id，此帳號即為分店管理者帳號。

目錄圖示 (icon)

分店在 yup 上列示時使用之圖片，空白則以 yup「無圖片」之圖示代表。

店面首頁圖示

分店在 yup 首頁所選用的背景與 logo，空白則以 yup「預設圖片」代表。
  • 左背景 (left_icon)：首頁左邊版面所選用之圖片。
  • logo (logo_icon)：分店所選用之 logo 圖片，空白則以 Facebook 粉絲頁之「大頭照」代表。
  • 右背景 (right_icon)：首頁右邊版面所選用之圖片。

郵遞區號 (zip_no)

非必要欄位。

縣市 (address1)

非必要欄位，選擇該分店所在的縣市，鄉鎮區內容會跟著連動。

鄉鎮區 (address2)

非必要欄位，選擇該分店所在的鄉鎮區。

聯絡地址 (address3)

非必要欄位，路街巷弄號樓等詳細地址。

市話號碼 (tel_no)

非必要欄位，輸入格式為 xx-xxxxxxxx。

傳真號碼 (fax_no)

非必要欄位，輸入格式為 xx-xxxxxxxx。

店老闆 (store_owner)

非必要欄位，各分店營收統計用。

Email(電腦) (mail_pc)

非必要欄位，指電腦上收發 email 的帳號。

Email(手機) (mail_mobile)

非必要欄位，指手機上收發簡訊的帳號，日本較為通行。

是否顯示 (display_yn)

是否顯示在 yup 首頁或店家類別中，預設為「是 (on)」。

營運中/休店 (open_yn)

此分店是否正常營運中，預設為「營運中 (on)」，若切換到「休店」，則此分店停止接受顧客訂購。

連結外部賣場的URL (link_url)

透過此外部 URL，顧客可以連結到其他網址。

是否接受信用卡 (credit_card_yn)

此分店是否接受信用卡，預設為「否 (off)」。

店舖公告事項 (store_notice)

此分店的公告事項。

初次網購附註 (first_order_comment)

此分店預設的提醒或通知事項，在新客戶第一次購買時顯示。

外送最低金額 (minimum_ammount)

此分店預設的外送最低金額，低於此金額，將不接受外送訂單。

類別 (uCatalogStore Entities)

此分店在 yup 所顯示的「類別標簽」，可複選。
